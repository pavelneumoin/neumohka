## 1. Идея в одной строке

Учитель оформляет подписку → получает доступ к **личной библиотеке Павла**: готовым урокам (презентация + рабочий лист + ответы), которые тот делает к каждому своему уроку и складывает в `Lessons\library\`.

Ключ — **бренд = личность**. Учителя покупают не у безликого сайта, а у конкретного автора, чьи материалы уже видели в VK.

## 2. Кому

- Учитель математики/информатики 8–11 класс
- 30–55 лет, преимущественно женщины
- Уже видел уроки в VK-группе Павла, оценил качество
- Не хочет каждый день делать слайды с нуля
- Готов платить 590 ₽ за экономию 5–10 часов в неделю

## 3. Как это работает с моей стороны

1. Делаю урок в `lessons-pipeline` (как сейчас)
2. После QA → `Lessons\library\` (как сейчас)
3. Из `library\` автоматически синкается на сайт
4. На VK выкладываю **анонс + первый слайд** (без PDF)
5. PDF лежит на сайте и открыт **только** подписчикам — кроме одного бесплатного раздела

## 4. Бизнес-модель

| Тариф | Цена | Что внутри |
|-------|------|------------|
| Месяц | **590 ₽** | Полный доступ |
| Квартал | 1490 ₽ | −16 % |
| Год | 4990 ₽ | −30 %, бонус: ранний доступ к новинкам |

- **Триал:** один раздел сайта открыт навсегда — это и есть «бесплатная воронка» вместо триала с картой. Учителя пробуют, потом подписываются.
- **Опция:** разовая покупка одного урока — 149 ₽

## 5. Источник контента

`E:\YA\YandexDisk\Lessons\library\` — уже ~50+ уроков, 8 разделов. Каждый урок:
- `main.pdf` — презентация
- `worksheet.pdf` — рабочий лист
- `answers.pdf` — ответы (когда есть)
- `post.txt` — готовый описательный текст для карточки сайта
- Метаданные: класс, предмет, тема

## 6. Ключевые экраны

1. **Лендинг** — личностный hero («Привет, я Павел…»), скриншоты, тарифы, FAQ, блок «обо мне»
2. **Каталог** — сетка карточек, фильтры (класс, предмет, тема, тип)
3. **Карточка урока** — превью первого слайда + кнопка «Скачать PDF»
4. **Тарифы и оплата** — 3 плана + ЮKassa
5. **Бесплатный раздел** — открыт навсегда, ключевая воронка
6. **Личный кабинет** — последние скачанные, статус подписки, история платежей
7. **Админка** — список уроков, синк с library/

## 7. Технологии

| Слой | Что | Почему |
|------|-----|--------|
| Frontend | Next.js 14 (App Router) + TypeScript | Как mainEGE, есть наработки |
| База | PostgreSQL + Prisma | То же |
| Auth | NextAuth (email + magic link) | Без паролей, проще для учителей |
| Платежи | ЮKassa | РФ-карты + СБП |
| Файлы | Yandex Object Storage (S3) | PDF в облаке |
| Хостинг | Vercel или Selectel VPS | Решим в спринте 4 |

## 8. Защита контента

PDF легко скопировать — DRM убьёт UX. Минимум на старте:
- **Watermark** с email подписчика на каждой странице PDF
- **Лимит скачиваний:** 50 уроков/мес на «Месяц», 200 на «Год»
- **Юридически:** оферта с запретом перепродажи

## 9. Этапы (спринты по 2 недели)

| # | Цель |
|---|------|
| 1 | Лендинг + каталог + карточка урока (без оплаты) + бесплатный раздел |
| 2 | Auth + подписка + оплата ЮKassa |
| 3 | Авто-синк из `library\` + админка + watermark на PDF |
| 4 | SEO + Telegram-бот для уведомлений + аналитика |
| 5 | Запуск, объявление в VK, первые 50 подписчиков |

**Цель к концу 5-го спринта (~2.5 месяца):** 50 подписчиков → 30 000 ₽/мес.
